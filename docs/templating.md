# Outreach Messaging

Outreach messages are generated by agentic LLM systems, not string templates. There is no per-account message template — the agents decide what to say from campaign context, the lead's stored firmographic profile, and (for follow-ups) the email conversation so far.

Two agents, both in `core/agents/` and both extending the shared `_outreach_base.j2` prompt:

- **Opener** (`email_opener.py`, prompt `email_opener.j2`) — composes one cold email at `READY_TO_EMAIL`: an `EmailDraft{subject, body, follow_up_hours}` sent once, parking the deal at `EMAILED`.
- **Follow-up** (`follow_up.py`, prompt `follow_up_agent.j2`) — the agentic loop on `EMAILED` deals: reads replies (IMAP) and returns a `FollowUpDecision{action, message?, outcome?, follow_up_hours}`. See [Follow-Up Agent](./follow_up_agent.md).

The prompts live in `core/templates/prompts/`. Shared context (product docs, campaign target, booking link, seller identity, and the lead/conversation fact summaries) is assembled in `core/agents/prompt.py`. See [Template Variables](./template-variables.md) for what each prompt receives.
